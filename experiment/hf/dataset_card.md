# litelite-benchmarks — corpora, benchmark pools, and candles for the verifier-reward fine-tunes

Everything needed to reproduce the numbers in the two companion models
(`prooflite-qwen3-0.6b`, `stratlite-qwen3-0.6b`) and the paper. Scoring is
deterministic; the models are not (sampling is stochastic).

## Contents

```
prooflite/
  corpus_seed.jsonl     174 verified prooflite programs (the cold-start set)
  results/              base + C5-C8 benchmark pools (256 each), benchmark.txt,
                        train_curve.log
stratlite/
  corpus_seed.jsonl     134 stratlite programs (the cold-start set)
  results/              base + C7 pools, benchmark.txt, train_curve.log
  candles/              the three pinned 1h kline windows + provenance README
```

## Provenance & license

- The **corpora and benchmark pools** are program source generated by the models
  / key-free agents, released Apache-2.0 with the kit.
- The **candles** (`stratlite/candles/*.csv`) are public Binance historical 1h
  klines (standard `data.binance.vision` schema), included as small fixed
  samples for reproducibility; see `stratlite/candles/README.md` for the exact
  windows and column layout. They are third-party public market data, not
  produced by this project.

## Reproduce

Scoring lives in the kit at <https://github.com/compusophy/litelite>. Build the
verifiers, then e.g. `p6 eval prooflite/results/c6.jsonl` (validity ladder) and
`s5 eval stratlite/results/c7.jsonl stratlite/candles/BTCUSDT-1h-2024-01.csv`
(compile + held-out gate-clear). Full commands are in each `results/README.md`.
